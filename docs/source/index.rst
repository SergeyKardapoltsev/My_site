Документация как код (Docs as Code)
===================================

===================================

В этом проекте я описал мой опыт применения Docs as Code
для создания документации и её публикации на сайте.

Docs as Code (документация как код) — это подход, при котором
документация создаётся, управляется и хранится так же,
как и программный код.

В проекте я использовал:

- генератор документации Sphinx;
- язык разметки ReStructuredText;
- систему контроля версий Git;
- сервис GitHub;
- сервис Read the Docs.

Sphinx преобразовывает текстовые документы в разные форматы.
Изначально его разработали для документирования языка программирования
Python. Затем стали применять и в других проектах. Sphinx использует
язык разметки reStructuredText.

ReStructuredText (сокращение: **ReST**) – облегчённый язык разметки.
Документы с разметкой ReST – это обычные текстовые файлы с расширением
**.rst**. Их можно открывать и редактировать в любом текстовом редакторе
(например, в Блокноте). ReST предоставляет много возможностей для
структурирования и форматирования документации.

Git – одна из самых популярных систем контроля версий документов. Это
программное обеспечение, которое позволяет отслеживать изменения в
программах, текстовых файлах, веб-сайтах. С помощью Git можно параллельно
работать нескольким пользователям над одним проектом, даже над
одним файлом. Git хранит историю изменений и сведения о том, кто и когда
вносил эти изменения в файл.

GitHub — сервис для хранения IT-проектов и совместной работы над ними
с использованием Git. Это сайт, куда можно загрузить файлы своего проекта
для обмена с другими людьми.

Read the Docs – сервис для бесплатного хостинга документации (хостинг –
услуга по хранению сайта). Этот сервис позволяет автоматически собирать
и публиковать документацию из репозиториев на GitHub и других аналогичных
сервисах. Документация может автоматически обновляться на сайте сразу
после добавления в репозиторий нового текста (кода).

Работа с каждым из перечисленных инструментов описана в отдельных
разделах моего проекта.
